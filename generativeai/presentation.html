âœ… Approach 3: Validated Generation

The Problem:

Model sometimes generates targets that don't exist in code!
Target: "for"
Code: Uses "while" instead ğŸ˜±

The Solution: Two-Stage Approach

# Stage 1: Model outputs components separately output = llm.generate(prompt) # Parse model output CODE: for(int i = 0; i < 5; i++) { ... } TARGET: for DISTRACTORS: while, do, if # Stage 2: Deterministic validation & question creation def create_validated_question(code, target, distractors): # 1. Verify target exists in code if target not in code: return None # Reject! # 2. Deterministically replace target with blank question_code = code.replace(target, "_____", 1) # 3. Shuffle options, track answer position options = [target] + distractors random.shuffle(options) answer = options.index(target) + 1 return {'question': question_code, 'answer': answer}
Result: 100% consistency guaranteed! Target always matches answer.
