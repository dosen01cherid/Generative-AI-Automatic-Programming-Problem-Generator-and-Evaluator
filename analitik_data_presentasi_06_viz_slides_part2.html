üîó Visualisasi Bivariat

Visualisasi bivariat menampilkan hubungan antara dua variabel. Tujuannya adalah untuk mengidentifikasi pola, korelasi, dan insight yang muncul dari interaksi antar variabel.

üéØ Tujuan Analisis Bivariat

  • Identifikasi Hubungan: Apakah ada korelasi positif, negatif, atau tidak ada?
  • Kekuatan Hubungan: Seberapa kuat hubungan antar variabel?
  • Pola Non-linear: Apakah hubungannya linear atau ada pola lain?
  • Outliers & Anomali: Identifikasi data yang tidak biasa
  • Segmentasi: Perbedaan pola antar grup/kategori

üìç 5. SCATTER PLOT - Hubungan Antar Variabel

Definisi dan Kegunaan

Scatter plot menampilkan nilai dari dua variabel numerik sebagai titik-titik dalam sistem koordinat 2D. Posisi setiap titik ditentukan oleh nilai variabel X (horizontal) dan Y (vertikal).

‚ú® Kapan Menggunakan Scatter Plot?

  • Mengeksplorasi hubungan antara 2 variabel numerik
  • Mengidentifikasi korelasi (positif, negatif, atau tidak ada)
  • Mendeteksi outliers dan anomali
  • Melihat clustering atau grouping dalam data
  • Validasi asumsi linear regression

Contoh: Hubungan Harga Satuan vs Kuantitas

Koefisien Korelasi Pearson

\( r = \frac{\sum_{i=1}^{n}(x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum_{i=1}^{n}(x_i - \bar{x})^2 \sum_{i=1}^{n}(y_i - \bar{y})^2}} \)

Range: -1 ‚â§ r ‚â§ +1

üìä Interpretasi Korelasi

|r| < 0.3 Lemah: Hubungan sangat kecil atau tidak ada

0.3 ‚â§ |r| < 0.7 Sedang: Hubungan moderat, ada pola tetapi variance tinggi

|r| ‚â• 0.7 Kuat: Hubungan jelas dan konsisten

Arah Korelasi:

  • r > 0: Positif - Kedua variabel bergerak searah
  • r < 0: Negatif - Kedua variabel bergerak berlawanan arah
  • r ‚âà 0: Tidak ada hubungan linear

‚ö†Ô∏è PENTING: Korelasi ‚â† Kausalitas

Correlation does not imply causation!

Hanya karena dua variabel berkorelasi TIDAK berarti satu menyebabkan yang lain.

  • Contoh misleading: Penjualan es krim berkorelasi dengan kasus tenggelam. Bukan karena es krim menyebabkan tenggelam, tapi karena keduanya meningkat di musim panas!
  • Spurious correlation: Korelasi palsu karena confounding variables
  • Perlu analisis lebih lanjut: Eksperimen, time-order, mechanism theory

üìà Pola yang Terlihat

  • Linear: Garis lurus
  • Exponential: Kurva naik cepat
  • Logarithmic: Kurva naik lambat
  • No pattern: Titik tersebar random
  • U-shaped: Pola parabola
  • Clusters: Kelompok terpisah

üí° Tips Scatter Plot

  • Gunakan transparency untuk overlap
  • Tambahkan trend/regression line
  • Color code untuk kategori
  • Size bubbles untuk variabel ke-3
  • Jitter untuk discrete data
  • Annotate interesting points
# Contoh Kode Python untuk Scatter Plot dengan Korelasi import matplotlib.pyplot as plt import numpy as np from scipy import stats # Generate sample data np.random.seed(42) x = np.random.uniform(500000, 3000000, 200) y = 5 - 0.000001 * x + np.random.normal(0, 0.5, 200) y = np.clip(y, 1, 5) # Calculate correlation correlation = np.corrcoef(x, y)[0, 1] plt.figure(figsize=(10, 6)) plt.scatter(x, y, alpha=0.6, c='purple', s=50) # Add trend line z = np.polyfit(x, y, 1) p = np.poly1d(z) plt.plot(x, p(x), "r--", alpha=0.8, linewidth=2, label='Trend Line') plt.title('Harga Satuan vs Kuantitas Terjual', fontsize=14, fontweight='bold') plt.xlabel('Harga Satuan (IDR)', fontsize=12) plt.ylabel('Kuantitas', fontsize=12) plt.grid(True, alpha=0.3) # Add correlation text plt.text(0.05, 0.95, f'Korelasi: r = {correlation:.3f}', transform=plt.gca().transAxes, fontsize=12, bbox=dict(boxstyle='round', facecolor='wheat', alpha=0.8)) plt.legend() plt.tight_layout() plt.show()

üìä 6. GROUPED BAR CHART - Perbandingan Multi-Kategori

Definisi dan Kegunaan

Grouped bar chart membandingkan nilai numerik antar kategori utama, dengan setiap kategori dipecah menjadi sub-kategori. Bar untuk sub-kategori dikelompokkan bersama untuk memudahkan perbandingan.

‚ú® Kapan Menggunakan Grouped Bar Chart?

  • Membandingkan sub-kategori dalam setiap kategori utama
  • Menampilkan data dengan 2 variabel kategorikal dan 1 numerik
  • Analisis segmentasi (contoh: penjualan per produk per region)
  • Time comparison across categories (contoh: Q1 vs Q2 per produk)
  • A/B testing results per segment

Contoh: Penjualan per Kategori dan Segmen Pelanggan

Grouped vs Stacked

Grouped Bar:

  • Mudah membandingkan nilai dalam grup
  • Lebih baik untuk perbandingan langsung
  • Butuh space horizontal lebih banyak

Stacked Bar:

  • Menampilkan total keseluruhan
  • Sulit bandingkan sub-kategori tengah
  • Lebih compact

üí° Tips Interpretasi

  • Bandingkan antar sub-kategori
  • Cari pola konsisten
  • Identifikasi outlier kategori
  • Perhatikan magnitude difference
  • Cross-category comparison
  • Look for opportunities

üéØ Insight Bisnis dari Grouped Bar

Yang Harus Dicari:

  1. Performa Terbaik: Kombinasi kategori-segmen dengan nilai tertinggi
  2. Gap Analysis: Perbedaan besar antar segmen dalam kategori sama
  3. Consistency: Apakah satu segmen selalu lebih baik di semua kategori?
  4. Opportunities: Kategori dengan performa rendah di segmen tertentu
  5. Strategic Focus: Mana yang perlu ditingkatkan vs dipertahankan
# Contoh Kode Python untuk Grouped Bar Chart import matplotlib.pyplot as plt import numpy as np import pandas as pd # Sample data data = { 'Electronics': {'Premium': 300, 'Regular': 250, 'Budget': 150}, 'Fashion': {'Premium': 280, 'Regular': 350, 'Budget': 200}, 'Home & Living': {'Premium': 220, 'Regular': 240, 'Budget': 120}, 'Beauty': {'Premium': 150, 'Regular': 180, 'Budget': 100}, 'Sports': {'Premium': 180, 'Regular': 220, 'Budget': 130} } df = pd.DataFrame(data).T # Plotting fig, ax = plt.subplots(figsize=(12, 6)) df.plot(kind='bar', ax=ax, width=0.8, color=['#667eea', '#f093fb', '#4facfe']) plt.title('Total Penjualan per Kategori dan Segmen Pelanggan', fontsize=14, fontweight='bold') plt.xlabel('Kategori Produk', fontsize=12) plt.ylabel('Total Penjualan (Juta IDR)', fontsize=12) plt.legend(title='Segmen Pelanggan', loc='upper right') plt.xticks(rotation=45, ha='right') plt.grid(axis='y', alpha=0.3) plt.tight_layout() plt.show()

üì¶ 7. BOX PLOT - Distribusi dan Outliers

Definisi dan Kegunaan

Box plot (atau box-and-whisker plot) adalah metode standar untuk menampilkan distribusi data berdasarkan five-number summary: minimum, Q1 (quartile pertama), median, Q3 (quartile ketiga), dan maximum. Sangat efektif untuk mengidentifikasi outliers.

Five-Number Summary

Minimum
Data terkecil
(atau Q1 - 1.5√óIQR)
Q1 (25%)
Quartile 1
25% data di bawah
Median (50%)
Quartile 2
Nilai tengah
Q3 (75%)
Quartile 3
75% data di bawah
Maximum
Data terbesar
(atau Q3 + 1.5√óIQR)

IQR (Interquartile Range) = Q3 - Q1

Outliers: Data < Q1 - 1.5√óIQR atau Data > Q3 + 1.5√óIQR

Contoh: Distribusi Penjualan per Kategori

üìä Cara Membaca Box Plot

1. Box (Kotak):

  • Bottom edge = Q1 (25th percentile)
  • Line di tengah = Median (50th percentile)
  • Top edge = Q3 (75th percentile)
  • Tinggi box = IQR (variability dari middle 50% data)

2. Whiskers (Garis):

  • Lower whisker = Q1 - 1.5√óIQR (atau minimum jika dalam range)
  • Upper whisker = Q3 + 1.5√óIQR (atau maximum jika dalam range)

3. Outliers:

  • Titik-titik di luar whiskers
  • Data yang unusually high atau low

üìà Interpretasi

Box Lebar:

  • High variability
  • Data spread out
  • Less consistency

Box Sempit:

  • Low variability
  • Data concentrated
  • More consistency

Median Position:

  • Di tengah = Symmetric
  • Ke atas = Negative skew
  • Ke bawah = Positive skew

üí° Kapan Gunakan

  • Membandingkan distribusi antar grup
  • Identifikasi outliers
  • Assess data spread & symmetry
  • Quality control
  • Exploratory data analysis
  • Robust terhadap outliers
  • Complement untuk histogram

‚ö†Ô∏è Keterbatasan Box Plot

  • Tidak menampilkan distribusi detail: Bimodal distribution terlihat seperti unimodal
  • Sample size matters: Dengan data kecil (<20), outlier detection kurang reliable
  • Tidak intuitif: Perlu pemahaman statistik untuk interpretasi
  • Alternatif: Violin plot menampilkan density distribution lebih jelas
# Contoh Kode Python untuk Box Plot import matplotlib.pyplot as plt import numpy as np # Generate sample data categories = ['Electronics', 'Fashion', 'Home & Living', 'Beauty', 'Sports'] data = [ np.random.normal(1500, 400, 100), np.random.normal(800, 200, 100), np.random.normal(1000, 300, 100), np.random.normal(300, 80, 100), np.random.normal(900, 250, 100) ] fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(14, 6)) # Box plot ax1.boxplot(data, labels=categories, patch_artist=True) ax1.set_title('Distribusi Penjualan per Kategori (Box Plot)', fontsize=13, fontweight='bold') ax1.set_xlabel('Kategori Produk') ax1.set_ylabel('Jumlah Penjualan (Ribu IDR)') ax1.tick_params(axis='x', rotation=45) ax1.grid(axis='y', alpha=0.3) # Violin plot (alternative) parts = ax2.violinplot(data, positions=range(1, 6), showmedians=True) ax2.set_xticks(range(1, 6)) ax2.set_xticklabels(categories, rotation=45, ha='right') ax2.set_title('Distribusi Penjualan per Kategori (Violin Plot)', fontsize=13, fontweight='bold') ax2.set_xlabel('Kategori Produk') ax2.set_ylabel('Jumlah Penjualan (Ribu IDR)') ax2.grid(axis='y', alpha=0.3) plt.tight_layout() plt.show()

üìä 8. DASHBOARD KOMPREHENSIF

Definisi dan Tujuan

Dashboard adalah kumpulan visualisasi yang terintegrasi untuk memberikan overview lengkap tentang data. Dashboard yang baik menceritakan story dan memudahkan pengambilan keputusan.

‚ú® Prinsip Dashboard yang Efektif

üìå Structure:

  • Logical flow (top-to-bottom, left-to-right)
  • Most important di top-left
  • Group related metrics

üé® Design:

  • Consistent color scheme
  • Balanced white space
  • Clear hierarchy

üìä Content:

  • Right charts for right data
  • No chart junk
  • Actionable insights

üéØ Purpose:

  • Clear objective
  • Know your audience
  • Tell a story

Contoh: E-Commerce Analytics Dashboard

2.8 hari

üéØ Dashboard Best Practices

  1. Start with KPIs: Tampilkan key metrics di atas (total revenue, conversions, etc.)
  2. Use appropriate charts: Jangan paksa semua data masuk, pilih yang relevan
  3. Consistent formatting: Color palette, fonts, sizes yang seragam
  4. Interactive elements: Filters, drill-downs untuk eksplorasi lebih dalam
  5. Update frequency: Real-time, daily, weekly - sesuai kebutuhan
  6. Mobile responsive: Pastikan readable di berbagai devices
  7. Performance: Load cepat, tidak terlalu banyak data sekaligus

‚ùå Dashboard Pitfalls yang Harus Dihindari

  • Too much information: Information overload, sulit fokus
  • Inconsistent metrics: Membingungkan, sulit compare
  • Poor chart choices: 3D pie charts, dual axis yang misleading
  • No context: Numbers tanpa benchmark atau comparison
  • Static only: Tidak bisa explore lebih dalam
  • Unclear purpose: Tidak jelas untuk siapa dan untuk apa
