Apa itu Data Analytics Lifecycle?

Data Analytics Lifecycle adalah sebuah framework yang menggambarkan tahapan-tahapan sistematis dalam mengeksekusi proyek analitik data, dari identifikasi masalah hingga implementasi solusi dalam produksi.

Mengapa Lifecycle Penting?

Struktur yang Jelas - Memberikan panduan langkah demi langkah
Mengurangi Risiko - Identifikasi masalah lebih awal
Komunikasi Efektif - Bahasa umum untuk tim
Iterative Process - Memungkinkan improvement berkelanjutan
Quality Assurance - Memastikan kualitas output
Resource Optimization - Efisiensi waktu dan biaya

Karakteristik Lifecycle

  • Iterative - Dapat kembali ke fase sebelumnya jika diperlukan
  • Flexible - Dapat disesuaikan dengan kebutuhan project
  • Collaborative - Melibatkan berbagai stakeholder
  • Data-Driven - Setiap keputusan berdasarkan data dan evidence
  • Goal-Oriented - Fokus pada pencapaian business objectives
üí° Key Insight: Lifecycle bukan proses linear, tetapi iterative - Anda bisa kembali ke fase sebelumnya berdasarkan findings atau feedback.

Manfaat Menggunakan Lifecycle Framework

Untuk Data Scientists

  • Clear roadmap
  • Best practices
  • Quality checkpoints

Untuk Business

  • Predictable timeline
  • Measurable progress
  • Better ROI

Untuk Organisasi

Discovery: Best Practices & Common Mistakes

‚úÖ Best Practices

1. Invest Time in Business Understanding

  • Schedule multiple stakeholder interviews
  • Shadow end users to understand workflows
  • Review existing reports and dashboards
  • Understand domain-specific terminology

2. Use Structured Problem Definition

5W1H Framework:

  • What: What is the business problem?
  • Why: Why is it important? Why now?
  • Who: Who is affected? Who benefits?
  • When: When did it start? When is deadline?
  • Where: Where does it occur?
  • How: How will analytics help?

3. Document Everything

  • Meeting notes with stakeholders
  • Assumptions and constraints
  • Data source details
  • Decisions made and rationale

‚ùå Common Mistakes to Avoid

1. Solution-First Approach

Mistake: "Let's use deep learning!" before understanding the problem

Impact: Over-engineering, wasted resources, misaligned solutions

Fix: Always start with problem definition, then choose appropriate tools

2. Unclear Success Criteria

Mistake: "Build a good model" without defining "good"

Impact: Scope creep, endless iterations, stakeholder disappointment

Fix: Define specific, measurable success criteria upfront

3. Ignoring Constraints

Mistake: Not identifying resource/data/time constraints early

Impact: Unrealistic plans, project delays, budget overruns

Fix: Document all constraints in project charter

4. Skipping Hypothesis Formation

Mistake: Diving into data exploration without hypotheses

Impact: Aimless analysis, p-hacking, spurious correlations

Fix: Formulate testable hypotheses based on domain knowledge

Missing Value Strategies

Deletion Methods

  • Listwise: Remove entire row
    df.dropna()
    ‚úì Simple, ‚úó Loses data
  • Pairwise: Use available data per analysis
    ‚úì Preserves data, ‚úó Inconsistent sample sizes
  • Column: Drop features with >40% missing
    df.dropna(axis=1, thresh=0.6*len(df))

Imputation Methods

  • Mean/Median/Mode:
    df.fillna(df.mean())
  • Forward/Backward Fill:
    df.fillna(method='ffill')
  • Interpolation:
    df.interpolate()
  • Predictive:
    Use ML model to predict missing values
  • Domain-specific:
    Business rules (e.g., 0 for first purchase)

Outlier Detection

IQR Method

\[IQR = Q_3 - Q_1\] \[\text{Lower Bound} = Q_1 - 1.5 \times IQR\] \[\text{Upper Bound} = Q_3 + 1.5 \times IQR\]

Values outside bounds considered outliers

Z-Score Method

\[z = \frac{x - \mu}{\sigma}\]

Typically |z| > 3 considered outlier

Python Example: Data Cleaning

import pandas as pd
import numpy as np

# Load data
df = pd.read_csv('customer_data.csv')

# Check missing values
print(df.isnull().sum())

# Handle missing values
df['age'].fillna(df['age'].median(), inplace=True)
df['income'].fillna(df.groupby('occupation')['income'].transform('median'), inplace=True)

# Remove duplicates
df.drop_duplicates(subset=['customer_id'], keep='first', inplace=True)

# Detect outliers using IQR
Q1 = df['purchase_amount'].quantile(0.25)
Q3 = df['purchase_amount'].quantile(0.75)
IQR = Q3 - Q1
lower_bound = Q1 - 1.5 * IQR
upper_bound = Q3 + 1.5 * IQR

# Cap outliers
df['purchase_amount'] = df['purchase_amount'].clip(lower_bound, upper_bound)

# Standardize categorical values
df['country'] = df['country'].replace({'US': 'USA', 'U.S.': 'USA', 'United States': 'USA'})

# Validate data
assert df['age'].min() >= 0, "Age cannot be negative"
assert df['age'].max() <= 120, "Age seems unrealistic"

3. Data Transformation

Scaling & Normalization

Technique Formula Range When to Use
Min-Max Normalization \(x' = \frac{x - x_{min}}{x_{max} - x_{min}}\) [0, 1] Neural networks, distance-based algorithms
Standardization (Z-score) \(z = \frac{x - \mu}{\sigma}\) Mean=0, Std=1 Algorithms assuming normal distribution
Robust Scaling \(x' = \frac{x - Q_2}{Q_3 - Q_1}\) Varies Data with outliers
Max Absolute Scaling \(x' = \frac{x}{|x_{max}|}\) [-1, 1] Sparse data

Encoding Categorical Variables

Label Encoding

Ordinal categories

Low ‚Üí 0
Medium ‚Üí 1
High ‚Üí 2

Use for: Ordinal data (education level, satisfaction rating)

One-Hot Encoding

Nominal categories

Color_Red: [1,0,0]

High-cardinality categories

Other Transformations

  • Log Transform: Handle skewed distributions
    \[x' = \log(x + 1)\]

    Useful for: Income, population, web traffic

  • Square Root Transform: Moderate skewness
    \[x' = \sqrt{x}\]
  • Box-Cox Transform: Make data more normal
    \[x' = \begin{cases} \frac{x^\lambda - 1}{\lambda} & \text{if } \lambda \neq 0 \\ \ln(x) & \text{if } \lambda = 0 \end{cases}\]
  • Binning/Discretization: Convert continuous ‚Üí categorical
    • Equal-width: Same interval size
    • Equal-frequency: Same number of samples per bin
    • Custom: Domain-specific boundaries (e.g., age groups)

Data Quality Management

"Garbage In, Garbage Out" - Data quality directly impacts model quality!

Data Quality Dimensions

1. Completeness

Extent to which data is not missing

\[\text{Completeness} = \frac{\text{Non-null values}}{\text{Total values}}\]

Target: >95% for critical fields

2. Accuracy

Data correctly represents reality

  • Validation rules
  • Cross-reference checks
  • Range verification

3. Consistency

Same data across sources matches

  • Format consistency
  • Value consistency
  • Temporal consistency

4. Timeliness

Data is up-to-date

  • Freshness metrics
  • Update frequency
  • Lag time

5. Validity

Data conforms to rules

  • Data type compliance
  • Format compliance
  • Business rule compliance

6. Uniqueness

No unintended duplicates

\[\text{Uniqueness} = \frac{\text{Unique records}}{\text{Total records}}\]

# Example validation checks
def validate_data(df):
    issues = []

    # Completeness checks
    if df['customer_id'].isnull().sum() > 0:
        issues.append("customer_id has missing values")

    # Range checks
    if (df['age'] < 0).any() or (df['age'] > 120).any():

    # Consistency checks
    if (df['order_date'] > df['ship_date']).any():

    # Uniqueness checks
    if df['transaction_id'].duplicated().any():
        issues.append("duplicate transaction_ids found")

    return issues

Handling Data Quality Issues

Decision Framework

When to Fix:

  • Issue affects >10% of records
  • Critical fields (IDs, target variable)
  • Systematic/recurring issues
  • Business-critical accuracy

When to Flag:

  • Minor issues (<5% of records)
  • Uncertain how to fix
  • Requires domain expert input

When to Remove:

  • Unfixable data corruption
  • Extreme outliers with no business meaning
  • Test/dummy records

Correlation Analysis

Pearson Correlation Coefficient

\[r = \frac{\sum (x_i - \bar{x})(y_i - \bar{y})}{\sqrt{\sum (x_i - \bar{x})^2 \sum (y_i - \bar{y})^2}}\]

Range: [-1, 1] where -1 = perfect negative, 0 = no correlation, 1 = perfect positive

Feature Selection Methods

1

Filter Methods

Statistical tests independent of ML algorithm

  • Variance Threshold: Remove low-variance features
  • Correlation: Remove highly correlated features (>0.9)
  • Chi-Square Test: For categorical features
  • ANOVA F-test: For continuous features
  • Mutual Information: Measures dependency
from sklearn.feature_selection import SelectKBest, f_classif

# Select top 10 features
selector = SelectKBest(score_func=f_classif, k=10)
X_selected = selector.fit_transform(X_train, y_train)
2

Wrapper Methods

Use ML algorithm to evaluate feature subsets

  • Forward Selection: Start with 0, add features one by one
  • Backward Elimination: Start with all, remove one by one
  • Recursive Feature Elimination (RFE): Iteratively remove least important
from sklearn.feature_selection import RFE
from sklearn.ensemble import RandomForestClassifier

estimator = RandomForestClassifier()
rfe = RFE(estimator, n_features_to_select=10)
X_selected = rfe.fit_transform(X_train, y_train)
3

Embedded Methods

Feature selection during model training

  • Lasso (L1 Regularization): Shrinks coefficients to zero
  • Tree-based Feature Importance: From Random Forest, XGBoost
  • Elastic Net: Combination of L1 and L2
from sklearn.ensemble import RandomForestClassifier

Key Metrics

Accuracy

\[\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}\]

Overall correctness

Use when: Balanced classes

Don't use when: Imbalanced data

Precision

\[\text{Precision} = \frac{TP}{TP + FP}\]

Of predicted positives, how many are correct?

Use when: False positives costly (spam filter)

Recall (Sensitivity)

\[\text{Recall} = \frac{TP}{TP + FN}\]

Of actual positives, how many did we catch?

Use when: False negatives costly (disease detection)

F1-Score

\[F1 = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}\]

Harmonic mean of precision and recall

Use when: Need balance, imbalanced data

Example Calculation

Cancer Detection Model

Results from 1000 patients:

  • TP = 85 (correctly detected cancer)
  • FP = 20 (false alarm, no cancer)
  • TN = 880 (correctly identified healthy)
  • FN = 15 (missed cancer cases)

Accuracy = (85 + 880) / 1000 = 96.5%

Precision = 85 / (85 + 20) = 81.0% (of predicted cancer, 81% actually have it)

Recall = 85 / (85 + 15) = 85.0% (caught 85% of cancer cases)

F1-Score = 2 √ó (0.81 √ó 0.85) / (0.81 + 0.85) = 82.9%

Interpretation: While accuracy is high (96.5%), we're missing 15% of cancer cases (FN=15). For medical diagnosis, we might prefer higher recall even at cost of lower precision.

ROC Curve & AUC

ROC (Receiver Operating Characteristic) Curve: Plot of True Positive Rate vs False Positive Rate at various threshold settings.

AUC (Area Under Curve): Single metric summarizing model performance across all thresholds.

\[\text{True Positive Rate (TPR)} = \frac{TP}{TP + FN} = \text{Recall}\] \[\text{False Positive Rate (FPR)} = \frac{FP}{FP + TN}\]

AUC Interpretation:

  • AUC = 1.0: Perfect classifier
  • AUC = 0.9-1.0: Excellent
  • AUC = 0.8-0.9: Good
  • AUC = 0.7-0.8: Fair
  • AUC = 0.5: Random (no better than coin flip)

Regression Metrics

Mean Absolute Error (MAE)

\[MAE = \frac{1}{n}\sum_{i=1}^{n}|y_i - \hat{y}_i|\]

Average absolute difference

Pros: Easy to interpret, same units as target

Cons: Doesn't penalize large errors heavily

Mean Squared Error (MSE)

\[MSE = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2\]

Average squared difference

Pros: Penalizes large errors

Cons: Squared units, sensitive to outliers

Root Mean Squared Error (RMSE)

\[RMSE = \sqrt{\frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2}\]

Square root of MSE

Pros: Same units as target, penalizes large errors

Most commonly used for regression

R¬≤ (R-Squared)

\[R^2 = 1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2}\]

Proportion of variance explained

Range: 0 to 1 (higher is better)

R¬≤ = 0.85 means model explains 85% of variance

Which Metric to Use?

Scenario Recommended Metric Reasoning
Balanced classification Accuracy, F1-Score Classes equally important
Imbalanced classification F1-Score, AUC-ROC Accuracy misleading with imbalance
Medical diagnosis (cancer) Recall (minimize FN) Missing cancer is worse than false alarm
Spam detection Precision (minimize FP) Blocking legitimate email is bad
Fraud detection F1-Score, AUC-ROC Balance between catching fraud and false alarms
Price prediction RMSE, MAE Regression problem
Stock price (outliers matter) RMSE Penalize large errors
Housing price (robust to outliers) MAE Less sensitive to outliers

Overfitting vs Underfitting

Understanding the bias-variance tradeoff is crucial for building models that generalize well.

The Three Scenarios

Underfitting

High Bias, Low Variance

  • Too simple model
  • Poor on training data
  • Poor on test data
  • Doesn't capture patterns

Train: 65%
Test: 63%

Good Fit

Balanced Bias-Variance

  • Right complexity
  • Good on training
  • Good on test
  • Generalizes well

Train: 88%
Test: 85%

Overfitting

Low Bias, High Variance

  • Too complex model
  • Excellent on training
  • Poor on test data
  • Memorized noise

Train: 99%
Test: 72%

Detecting Overfitting

üîç Warning Signs

Performance Indicators:

  • Large gap between train and test accuracy
  • Training loss decreases, validation loss increases
  • Very high accuracy on training (>95% often suspicious)
  • Poor performance on new/unseen data

Model Indicators:

  • Model too complex for dataset size
  • Too many features (features > samples)
  • No regularization applied
  • Training for too many epochs

Solutions to Overfitting

1

Get More Training Data

Most effective but often not feasible

  • Collect more samples
  • Data augmentation (images: rotate, flip, crop)
  • Synthetic data generation
2

Reduce Model Complexity

  • Use simpler algorithm
  • Reduce number of features
  • Decrease polynomial degree
  • Prune decision trees (limit depth)
# Example: Limit tree depth
model = DecisionTreeClassifier(max_depth=5)  # Instead of unlimited depth
3

Regularization

Add penalty for model complexity

L1 Regularization (Lasso)

\[\text{Loss} = \text{MSE} + \lambda \sum_{i=1}^{n}|w_i|\]

Drives some weights to exactly zero (feature selection)

L2 Regularization (Ridge)

\[\text{Loss} = \text{MSE} + \lambda \sum_{i=1}^{n}w_i^2\]

Shrinks weights but doesn't eliminate features

from sklearn.linear_model import Ridge, Lasso

# Ridge regression with regularization

# Lasso regression
model = Lasso(alpha=0.1)

Solutions to Underfitting

Increase Model Capacity

  • Use more complex algorithm: Linear ‚Üí Polynomial ‚Üí Neural Network
  • Add more features: Feature engineering, polynomial features
  • Increase model size: More layers, more neurons
  • Train longer: More epochs
  • Reduce regularization: Lower Œª value

Bias-Variance Tradeoff

Total Error Decomposition

\[\text{Total Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Error}\]

Bias: Error from wrong assumptions (underfitting)

Variance: Error from sensitivity to training data (overfitting)

Irreducible Error: Noise in data (cannot reduce)

Status: Good generalization!

Model: Deep neural network with 100+ features (including noise)

Result: Train R¬≤ = 0.99, Test R¬≤ = 0.72

Problem: Memorized training data including noise

Solution: Regularization, feature selection, simpler model

  • Small train-test gap (<5%)
  • Validation loss decreases with training loss
  • Consistent performance across CV folds
  • Good performance on new data
‚ùå Red Flags
  • Train accuracy 99%, Test accuracy 70%
  • Validation loss increasing while train decreases
  • High variance across CV folds
  • Works on training, fails in production

Create Prediction API (Flask Example)

from flask import Flask, request, jsonify
import joblib
import numpy as np

app = Flask(__name__)

# Load model at startup
model = joblib.load('model_v1.0.pkl')
scaler = joblib.load('scaler.pkl')

@app.route('/predict', methods=['POST'])
def predict():
    try:
        # Get data from request
        data = request.json
        features = np.array(data['features']).reshape(1, -1)

        # Preprocess
        features_scaled = scaler.transform(features)

        # Predict
        prediction = model.predict(features_scaled)[0]
        probability = model.predict_proba(features_scaled)[0]

        # Return result
        return jsonify({
            'prediction': int(prediction),
            'probability': float(probability[1]),
            'model_version': '1.0'
        })

    except Exception as e:
        return jsonify({'error': str(e)}), 400

@app.route('/health', methods=['GET'])
def health():
    return jsonify({'status': 'healthy', 'model_loaded': True})

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)

Model Drift Detection

Model Drift: When model performance degrades over time because the relationship between features and target changes.

Data Drift (Covariate Shift)

Input feature distributions change

Example: E-commerce customer age distribution shifts from 30-40 to 20-30

Detection:

  • Compare feature distributions: train vs. production
  • Statistical tests (KS test, Chi-square)
  • Population Stability Index (PSI)

Concept Drift

Relationship between X and Y changes

Example: Customer behavior changes post-pandemic

Detection:

  • Monitor prediction accuracy over time
  • Compare actual vs. predicted outcomes
  • Rolling window evaluation

Monitoring Feature Drift with PSI

import numpy as np

def calculate_psi(expected, actual, bins=10):
    """Calculate Population Stability Index"""
    # Bin the data
    breakpoints = np.percentile(expected, np.linspace(0, 100, bins+1))

    expected_percents = np.histogram(expected, breakpoints)[0] / len(expected)
    actual_percents = np.histogram(actual, breakpoints)[0] / len(actual)

    # Avoid division by zero
    expected_percents = np.where(expected_percents == 0, 0.0001, expected_percents)
    actual_percents = np.where(actual_percents == 0, 0.0001, actual_percents)

    # Calculate PSI
    psi = np.sum((actual_percents - expected_percents) *
                 np.log(actual_percents / expected_percents))

    return psi

# Example usage
training_ages = np.random.normal(35, 10, 10000)
production_ages = np.random.normal(28, 12, 1000)  # Shifted distribution

psi = calculate_psi(training_ages, production_ages)
print(f"PSI: {psi:.4f}")

# Interpretation:
# PSI < 0.1: No significant change
# 0.1 <= PSI < 0.2: Moderate change, investigate
# PSI >= 0.2: Significant change, retrain model
